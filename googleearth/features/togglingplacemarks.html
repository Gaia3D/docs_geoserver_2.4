Toggling Placemarks¶

Vector Placemarks¶

GeoServer가 벡터 데이터셋에 대해 KML을 생산하는 경우, 생성하는 모든 피처에 데이터에서 나온 정보를 부여합니다. Google 어스에서 벡터 피처를 클릭하면 이 정보를 담은 팝업창이 뜹니다. 이것을 위치표시(placemark)라고 합니다. 기본적으로 위치표시는 속성값을 표출하는 간단한 목록이지만, Freemarker 템플릿을 이용해서 해당 정보를 사용자 지정할 수 있습니다.

(보안 문제 때문이든 단순히 깨끗한 사용자 인터페이스를 원하기 때문이든 간에) 피처를 클릭해도 이 정보가 표출되지 않기를 바랄 경우 이 기능을 비활성화할 수 있습니다. 그렇게 하려면 속성을 끄게 만드는 kmattr 파라미터를 KML 요청에 추가하십시오.

kmattr 파라미터의 구문은 다음과 같습니다.

format_options=kmattr:[true|false]

kmattr 파라미터가 “형식 옵션(format option)”이기 때문에 일반적인 키(key)/값 쌍과는 다음과 같이 구문이 약간 다르다는 점을 유의하십시오.

http://localhost:8080/geoserver/wms/kml?layers=topp:states&format_options=kmattr:false

Raster Placemarks¶

기본적으로 위치표시가 활성화 되어 있는 벡터 피처와 달리, 피처의 래스터 이미지의 경우 위치표시가 기본적으로 비활성화되어 있습니다. 이 기능을 활성화하려면 KML 요청에 kmplacemark 형식 옵션을 사용하면 됩니다. 이 옵션의 구문은 앞의 kmattr 형식 옵션과 유사합니다.

format_options=kmplacemark:[true|false]

예를 들어 KML 반영자를 사용할 경우 다음과 같은 구문을 쓰게 됩니다.

http://localhost:8080/geoserver/wms/kml?layers=topp:states&format_options=kmplacemark:true
