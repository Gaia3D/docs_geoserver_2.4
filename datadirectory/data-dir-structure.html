Structure of the Data Directory¶

이 섹션에서 GeoServer의 데이터 디렉터리에 대한 개요를 설명합니다.

일반적으로 데이터 디렉터리 설정 파일은 직접 편집해서는 안 되기 때문에, 이 섹션을 GeoServer 설정 정보에 대한 완전한 자료로 삼아서는 안 됩니다. 대신 Web Administration Interface를 통해 설정 정보를 수동으로 열람하고 수정할 수 있으며, 규칙에 따르는 접속 및 조작을 위해서는 REST configuration API를 사용해야 합니다.

사용자가 수정할 수 있는 내용을 담고 있는 디렉터리에는 logs, palettes, templates, user-projection, www가 있습니다.

다음 그림은 GeoServer의 데이터 디렉터리 구조를 보여줍니다.

<data_directory>/

   global.xml
   logging.xml
   wms.xml
   wfs.xml
   wcs.xml

   data/
   demo/
   geosearch/
   gwc/
   layergroups/
   logs/
   palettes/
   plugIns/
   security/
   styles/
   templates/
   user_projections/
   workspaces/
     |
     +- workspace dirs...
        |
        +- datastore dirs...
           |
           +- layer dirs...
   www/

The .xml files¶

최상위(top-level) .xml 파일들은 서버 인스턴스가 사용하는 서비스들 및 다양한 일반 옵션들에 대한 정보를 담고 있습니다.

파일 설명
global.xml 모든 서비스에 공통적인 연락처 정보, JAI 설정, 캐릭터셋, 세부 정보 수준(verbosity) 등의 설정 정보를 담고 있습니다.
logging.xml 로그 수준, 로그 파일 위치, 로그를 표준 출력(stdout)할지 여부 등 로그 파라미터를 설정합니다.
wcs.xml WCS 서비스를 위한 서비스 메타데이터 및 여러 설정 정보를 담고 있습니다.
wfs.xml WFS 서비스를 위한 서비스 메타데이터 및 여러 설정 정보를 담고 있습니다.
wms.xml WMS 서비스를 위한 서비스 메타데이터 및 여러 설정 정보를 담고 있습니다.

workspaces¶

작업공간(workspaces) 디렉터리는 GeoServer가 생산한 레이어의 메타데이터를 담고 있습니다. 이 디렉터리에는 정의된 각각의 작업공간을 위한 디렉터리를 담고 있습니다. 각 작업공간 디렉터리는 해당 작업공간에 정의된 데이터 저장소(datastores)들을 위한 디렉터리들을 가지고 있습니다. 각 데이터 저장소 디렉터리는 해당 데이터 저장소에 정의된 레이어들을 위한 디렉터리들을 가지고 있습니다. 각 레이어 디렉터리는 layer.xml 파일과, 해당 레이어가 래스터(raster) 아니면 벡터(vector) 데이터셋을 표현하느냐에 따라 coverage.xml 아니면 featuretype.xml 파일을 담고 있습니다.

data¶

data 디렉터리는 레이어로 서비스되고 있는 파일 기반 지리공간(geospatial) 데이터셋들을 저장하기 위해 사용될 수 있습니다. (이 디렉터리를 주 “GeoServer data directory”와 혼동해서는 안 됩니다.) 이 디렉터리는 보통 shapefile과 래스터 파일을 저장하기 위해 사용되지만, 파일 기반 데이터라면 모두 저장할 수 있습니다.

data 디렉터리에 데이터 파일을 저장하면 휴대성·이식성(portability)이라는 장점을 가지게 됩니다. C:\gis_data\foo.shp 경로의 데이터 디렉터리에 외부 저장된 shapefile을 가정해보면, catalog.xml 파일 내부에서 이 shapefile에 대한 datastore 값은 다음과 같이 표현될 것입니다.

<datastore id="foo_shapefile">
   <connectionParams>
     <parameter name="url" value="file://C:/gis_data/foo.shp" />
   </connectionParams>
 </datastore>

이제 다른 호스트에서 실행 중인 GeoServer에 이 데이터 디렉터리를 복사(port)하려 한다고 가정해보십시오. 다른 호스트에 C:\gis_data\foo.shp 파일 경로가 없을 확률이 높습니다. 따라서 다른 호스트의 동일한 경로에 파일을 복사하든지, 아니면 새로운 경로를 반영하기 위해 catalog.xml 파일을 편집해야 합니다.

그런데 foo.shp 파일을 data 디렉터리에 저장하면 이런 문제가 생기지 않습니다. 이럴 경우 catalog.xml 파일의 datastore 값은 다음과 같습니다.

<datastore id="foo_shapefile">
  <connectionParams>
    <parameter name="url" value="file:data/foo.shp"/>
  </connectionParams>
</datastore>

value 속성이 data 디렉터리를 따라 변경되었습니다. 이런 위치 독립성(location independence) 덕분에 data 디렉터리를 통째로 다른 호스트에 복사하면 다른 설정 변경 없이도 바로 사용할 수 있게 됩니다.

demo¶

demo 디렉터리는 Sample Request Tool(http://localhost:8080/geoserver/demoRequest.do)에서 확인할 수 있는 sample requests에 대한 정보를 담고 있습니다. 자세한 내용은 Demos 페이지에서 확인하십시오.

geosearch¶

geosearch 디렉터리는 KML 파일의 출력 범위 기준(regionation)에 대한 정보를 담고 있습니다. 자세한 내용은 KML Regionation 페이지에서 확인하십시오.

gwc¶

gwc 디렉터리는 GeoWebCache 서비스가 생성한 캐시(cache)를 보관합니다.

layergroups¶

layergroups 디렉터리는 정의된 레이어 그룹(layergroup)에 대한 설정 정보를 담고 있습니다.

logs¶

logs 디렉터리는 로그 프로파일(logging profiles)과 기본 geoserver.log 로그 파일에 대한 설정 정보를 담고 있습니다. Advanced log configuration 페이지도 참조하십시오.

palettes¶

palettes 디렉터리는 미리 계산된(pre-computed) 이미지 팔레트(Image Palettes)를 저장하는 데 사용됩니다. GeoServer WMS가 생산된 이미지의 질을 유지하면서도 용량을 줄이기 위한 한 방편으로 이미지 팔레트를 사용합니다. Paletted Images 페이지도 참조하십시오.

security¶

security 디렉터리는 GeoServer의 보안 서브시스템을 설정하는 데 사용되는 파일들을 담고 있습니다. 접속 역할(access roles)을 정의하는 일련의 속성(property) 파일도 포함되는데, 이 속성 파일은 각 역할이 접속할 수 있도록 인증받은 서비스 및 데이터도 함께 정의하고 있습니다. 자세한 내용은 Security 섹션을 참조하십시오.

styles¶

styles 디렉터리는 GeoServer WMS가 사용하는 스타일 정보를 담은 Styled Layer Descriptor(SLD) 파일을 담고 있습니다. 이 디렉터리 안의 catalog.xml 파일 내부에는 각각 다음과 같은 값이 존재합니다.

<style id="point_style" file="default_point.sld"/>

스타일과 SLD에 대한 자세한 정보는 Styling 섹션을 참조하십시오.

templates¶

templates 디렉터리는 GeoServer의 templating 서브시스템이 사용하는 파일을 담고 있습니다. 템플릿은 다양한 GeoServer 작업의 산출물을 사용자 지정하는 데 사용됩니다. Freemarker Templates 페이지도 참조하십시오.

user_projections¶

user_projections 디렉터리는 사용자 지정 공간 참조 시스템(custom spatial reference systems)을 정의하는 데 사용되는 epsg.properties 파일을 담고 있습니다. 이 파일은 공식 EPSG 데이터베이스의 일부가 아닙니다. Custom CRS Definitions 페이지도 참조하십시오.

www¶

www 디렉터리는 GeoServer가 일반적인 웹 서버처럼 파일들을 서비스할 수 있도록 해줍니다. 이 디렉터리의 내용물은 http:/<host:port>/geoserver/www 주소에서 서비스됩니다. 정식 웹 서버를 대체할 만큼은 아니지만, 클라이언트 중심(client-side) 매핑 응용 프로그램을 서비스하는 데 유용합니다. Serving Static Files 페이지도 참조하십시오.
