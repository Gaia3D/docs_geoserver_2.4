Authentication providers¶

다음은 GeoServer에서 사용할 수 있는 인증 제공자입니다.

  • 사용자/그룹 서비스를 대상으로 사용자명/비밀번호를 인증
  • LDAP 서버를 대상으로 하는 인증
  • JDBC를 통해 데이터베이스에 연결하여 인증

Username/password authentication¶

사용자명/비밀번호 인증은 기본 인증 제공자입니다. 인증 작업에 사용자/그룹 서비스를 이용합니다.

단순하게 (기본 인증 요청 같은) 들어오는 요청에서 사용자명/비밀번호를 받아, 사용자/그룹 서비스에서 사용자 정보를 로드해서 자격 증명을 확인합니다.

LDAP authentication¶

LDAP 인증 제공자는 LDAP(Lightweight Directory Access Protocol) 서버를 대상으로 인증 작업을 할 수 있습니다. 이 제공자는 들어오는 요청에서 사용자명/비밀번호를 받아 해당 자격 증명으로 LDAP 서버에 연결을 시도합니다.

주석

현재 버전은 LDAP Bind 인증만 지원합니다.

Role assignment¶

LDAP 제공자는 인증된 사용자에 대해 롤 할당(role assignment)을 위한 다음 2가지 옵션을 제공합니다.

  • 사용자의 LDAP 그룹을 롤로 변환
  • 사용자/그룹 서비스를 이용

다음은 첫번째 옵션을 보여주는 LDAP 데이터베이스입니다.

dn: ou=people,dc=acme,dc=com
objectclass: organizationalUnit
ou: people

dn: uid=bob,ou=people,dc=acme,dc=com
objectclass: person
uid: bob

dn: ou=groups,dc=acme,dc=com
objectclass: organizationalUnit
ou: groups

dn: cn=workers,ou=groups,dc=acme,dc=com
objectclass: groupOfNames
cn: users
member: uid=bob,ou=people,dc=acme,dc=com

이 시나리오에서 bob이라는 uid를 가진 사용자를 정의하고, bob이 속해 있는 workers라는 group도 정의합니다. 인증 작업 후, ROLE_WORKERS롤이 bob에게 할당됩니다. 롤 명칭은 ROLE_과 그룹명을 대문자로 표시한 문자열을 연결해서 만들어집니다.

주석

익명으로 LDAP 서버를 검색할 수 없을 경우, 오류를 발생시키지 않으려면 bindBeforeGroupSearch 옵션을 활성화해야 합니다.

사용자/그룹 서비스를 이용할 경우, 인증해야 하는 사용자를 사용자/그룹 서비스에 쿼리하고, 사용자/그룹 서비스와 활성화된 롤 서비스 모두 롤 할당 작업을 수행합니다. 이 옵션을 사용할 경우 사용자/그룹 서비스 데이터베이스에 저장된 사용자의 비밀번호를 무시합니다.

Secure LDAP connections¶

서버에 보안된 LDAP 연결을 생성하는 데 2가지 방법이 있습니다. 첫번째는 Server URL의 일부인 ldaps 프로토콜을 이용해서 직접 보안 연결을 설정하는 방법입니다. 이때 일반적으로 연결 포트를 389 포트가 아니라 636 포트로 변경해야 합니다.

두번째는 STARTTLS(Transport Layer Security)를 이용해서 비보안 연결을 보안 연결로 변경하는 방법입니다. 이 방법은 389 포트 상에서 “ldap” 프로토콜을 이용하는 비보안 URL에 쓸 수 있습니다. 이 옵션을 이용하려면 Use TLS 플래그를 설정해야 합니다.

경고

연결에 TLS를 이용하면 GeoServer가 LDAP 연결을 풀링할 수 없습니다. 즉 각 인증 작업마다 새로운 LDAP 연결을 생성했다가 종료해야 하는데, 성능 저하를 일으킬 수 있습니다.

JDBC authentication¶

JDBC 인증 제공자는 JDBC를 통해 데이터베이스에 연결해서 인증 작업을 수행합니다.

이 제공자는 들어오는 요청에서 사용자명/비밀번호를 받아 해당 자격 증명으로 데이터베이스 연결을 생성하려 시도합니다. 인증을 성공적으로 마친 후, 제공자가 선택적으로 사용자/그룹 서비스를 이용해서 사용자 정보를 로드할 수도 있습니다. 이 맥락에서 사용자/그룹 서비스는 비밀번호 확인이 아니라 롤 할당 작업만을 위해 쓰입니다.

주석

비밀번호를 확인하는 데 사용자/그룹 서비스를 이용하려면 Username/password authentication 섹션을 참조하십시오.
