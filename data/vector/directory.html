Directory of spatial files¶

디렉터리 저장소(directory store)는 GeoServer에 복수의 shapefile을 자동으로 로드합니다. 복수의 shapefile을 담고 있는 디렉터리를 로드하면, 각 shapefile이 자동적으로 GeoServer에 추가됩니다.

주석

GeoServer가 shapefile 포맷을 충분히 지원하기는 하지만, 데이터 생산 환경에서 추천할 만한 포맷은 아닙니다. PostGIS 같은 데이터베이스들이 데이터 생산에 더 적합하며, 더 나은 성능과 확장성을 제공하고 있습니다. 자세한 내용은 Running in a Production Environment 섹션을 참조하십시오.

Adding a directory¶

디렉터리를 추가하려면 Stores ‣ Add a new store ‣ Directory of spatial files 메뉴로 이동하십시오.

저장소로서 공간 파일 디렉터리를 추가하기

옵션 설명
Workspace 저장소를 담기 위한 작업공간의 명칭을 선택합니다. 해당 저장소의 shapefile에서 생성될 레이어명의 접두사(prefix)가 됩니다.
Data Source Name GeoServer가 인식할 저장소명을 설정합니다.
설명 디렉터리 저장소를 설명합니다.
Enabled 저장소를 활성화합니다. 체크를 해제할 경우 어떤 shapefile의 데이터도 서비스되지 않습니다.
URL 디렉터리 위치를 설정합니다. 절대 경로(예시: file:C:\Data\shapefile_directory)가 될 수도 있고 데이터 디렉터리에 대한 상대 경로(예시: file:data/shapefile_directory가 될 수도 있습니다.
namespace 저장소와 연결될 Namespace를 설정합니다. 이 항목은 작업공간 명칭에 따라 변경됩니다.

설정이 끝나면 Save 버튼을 클릭하십시오.

Configuring shapefiles¶

디렉터리 저장소에 담긴 모든 shapefile은 디렉터리 저장소의 일부로서 로드될 것입니다. 하지만 GeoServer가 서비스하려면 각각 새로운 레이어로 설정되어야 합니다. 새로운 레이어를 추가·편집하는 방법에 대해서는 Layers 페이지를 참조하십시오.
